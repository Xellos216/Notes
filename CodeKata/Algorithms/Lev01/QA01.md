## ✅ Q1: Python에서 'SyntaxError: invalid syntax' 오류 원인

**문제 상황**:  
Python 코드에서 `elif angle == 180:` 구문 직전에 `elif 90 < angle < 180:` 이 있었고,  
`answer = 3` 또는 `answer = 4`를 설정한 후 `return answer` 로 마무리했지만,  
**SyntaxError**가 발생했다.

**원인 분석**:  
코드에서 `answer = 3`, `answer = 4` 뒤의 `return answer` 문이 `if-elif-else` 블록 외부에 있어야 하는데  
들여쓰기가 일치하지 않거나, 특정 줄에 문법적인 실수가 있는 경우 발생할 수 있다.  
하지만 실제 오류는 Python 코드 자체의 문제보다, 해당 플랫폼에서 복붙으로 인해 줄 끝 세미콜론이나  
문법 파편이 섞였을 때 종종 발생한다.

**해결 방법**:  
모든 줄 끝 세미콜론 제거 및 **Python에서 불필요한 세미콜론 사용 금지** 확인.  
또한 코드 들여쓰기를 정확히 맞춰야 한다.

## ✅ Q2: Java에서 'bad operand types for binary operator' 오류

**문제 상황**:  
`if (0 < angle < 90)` 형태로 조건문을 작성했을 때 Java에서 컴파일 오류가 발생함.

**원인 분석**:  
Java는 Python처럼 **연속된 비교 (0 < angle < 90)** 를 지원하지 않음.  
`0 < angle` 의 결과는 `boolean` 이고, 그 후 `boolean < 90` 이 되는 구조는 Java에서 허용되지 않음.

**해결 방법**:  
비교 연산을 `if (0 < angle && angle < 90)` 형식으로 **논리 연산자 `&&`로 나누어** 작성해야 함.

## ✅ Q3: Java 코드에서 'variable might not have been initialized'

**문제 상황**:  
`int answer;` 선언 후 조건문 내에서만 값을 할당하고 `return answer;` 을 실행했을 때 오류 발생.

**원인 분석**:  
Java는 지역 변수에 대해 **명시적인 초기화 없이 사용하면 오류 발생**.  
특히 조건문 내에서만 초기화될 수 있으면, Java 컴파일러는 **“해당 분기를 안 탈 수도 있다”고 판단**하여 오류를 낸다.

**해결 방법**:  
`int answer = 0;` 과 같이 **기본값을 주고 시작**하면 해결됨.

## ✅ Q4: Kotlin에서 for문 문법 오류 ("expecting 'in'" 등)

**문제 상황**:  
Kotlin에서 다음과 같은 Java 스타일 for문을 작성할 경우 오류 발생:

```kotlin
for (int i = 2; i <= n; i += 2) { ... }
```

**원인 분석**:  
Kotlin은 Java의 `for (int i = ... )` 문법을 **지원하지 않음**.  
Kotlin의 반복문은 `for (i in 0 until n)` 또는 `for (i in 2..n step 2)` 와 같은 **컬렉션 기반 루프**를 사용함.

**해결 방법**:  
예를 들어 짝수만 순회할 때는 다음처럼 작성해야 함:

```kotlin
for (i in 2..n step 2) {
    answer += i
}
```

## ✅ Q5: Python 코드에서 "C 스타일 for문"을 사용했을 때 오류

**문제 상황**:  
Python 코드에서 `for (int i = 1; i <= n; i++)` 와 같은 문법을 사용하면 오류 발생.

**원인 분석**:  
Python은 **C/Java 스타일의 for 루프를 지원하지 않음**.  
`for i in range(...)` 형태만 존재함.

**해결 방법**:  
다음처럼 Python 스타일로 바꿔야 한다:

```python
for i in range(1, n + 1):
    ...
```

## ✅ Q6: Python의 range(2, n + 1, 2)는 짝수만 도는가?

**문제 상황**:  
`range(2, n + 1, 2)` 를 사용했을 때 짝수만 순회하는 게 맞는지 의문이 생김.

**원인 분석**:  
`range(start, stop, step)` 에서 `step = 2` 이고 `start = 2`면,
2부터 시작해서 2씩 증가 → 즉 2, 4, 6, ..., n 이하의 짝수만 순회함.

**결론**:  
`range(2, n + 1, 2)`는 **2부터 n까지의 짝수만 순회**하게 된다. 올바른 짝수 누적 방식임.

## ✅ Q7: Java에서 for문으로 짝수만 더하려면?

**답변 요약**:

```java
for (int i = 2; i <= n; i += 2) {
    answer += i;
}
```

- `i = 2`: 짝수 시작
- `i += 2`: 2씩 증가 → 짝수만 순회

## ✅ Q8: Kotlin에서 "expecting 'in'" 오류 이유

**원인**: Kotlin의 for문은 **범위 기반 문법만 사용 가능**하기 때문.  
즉 `for (int i = 0; ... )`는 허용되지 않고, 다음과 같이 작성해야 함:

```kotlin
for (i in 0 until n) {
    ...
}
```

## ✅ Q9: Python은 왜 n 제한 조건을 코드에 안 써도 되는가?

**문제 상황**:  
제한사항에 `0 < n <= 1000` 이 있었는데, 왜 코드상에서 별도 체크 안 해도 되는가?

**답변**:  
해당 조건은 **문제 출제자가 보장하는 입력값의 범위**이기 때문에  
별도로 `if n > 1000:` 같은 방어 코드 없이 작성해도 무방함.  
테스트 케이스가 유효 입력값 내에서만 주어진다는 전제이기 때문.
