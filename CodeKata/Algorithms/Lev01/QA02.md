## Q11: Python에서 `toDouble()`은 무엇인가요?

Python에는 Kotlin처럼 `.toDouble()`이라는 메서드는 존재하지 않습니다. Kotlin에서는 `Int`나 `String` 타입의 값을 `Double`로 변환할 때 `.toDouble()`이라는 메서드를 사용하지만, Python에서는 기본 타입 변환 함수 `float()`를 사용합니다.

### 예시:
```python
x = 42
y = float(x)  # Kotlin의 x.toDouble()과 동일한 효과
```

## Q12: Python에서 정수와 실수 연산은 어떻게 처리되나요?

Python은 동적 타입 언어이기 때문에 정수(`int`)와 실수(`float`)가 함께 연산되면 자동으로 실수로 형 변환됩니다.

### 예시:
```python
result = 5 / 2  # 2.5 (자동으로 float)
```

## Q13: Kotlin에서 자료형을 명시하지 않으면 어떻게 되나요?

Kotlin은 기본적으로 **타입 추론** 기능을 제공합니다. 즉, 초기화하는 값을 보고 자료형을 자동으로 추론합니다.

### 예시:
```kotlin
val x = 10   // Int로 추론
val y = 3.14 // Double로 추론
```

명시적으로 자료형을 지정하고 싶다면 다음처럼 작성합니다:
```kotlin
val x: Int = 10
val y: Double = 3.14
```

## Q14: Kotlin에서 `toDouble()` 앞의 `to`는 무슨 의미인가요?

`toDouble()`에서 `to`는 '변환하다'는 의미입니다. `toInt()`, `toFloat()`, `toString()` 등으로 다양하게 쓰이며, Kotlin 표준 라이브러리의 형변환 함수입니다.

### 요약:
- `toDouble()` → Double 타입으로 변환
- `toInt()` → 정수(Int) 타입으로 변환
- `toString()` → 문자열(String)로 변환

## Q15: Python 에러 "SyntaxError: invalid syntax"는 왜 발생했나요?

Python에서 C처럼 `int i = 0;` 과 같은 선언문을 쓸 수 없습니다. Python은 다음과 같이 작성해야 합니다:

### 잘못된 코드 (C 스타일):
```python
for (int i = 1; i <= n; i++):
```

### 올바른 Python 코드:
```python
for i in range(1, n + 1):
```

## Q16: Java 코드에서 `bad operand types for binary operator` 에러는 왜 발생했나요?

Java에서는 `if (0 < angle < 90)`처럼 Python 스타일의 범위 비교를 사용할 수 없습니다. 각 비교를 **명시적으로** 작성해야 합니다:

### 잘못된 예:
```java
if (0 < angle < 90)
```

### 올바른 예:
```java
if (0 < angle && angle < 90)
```

## Q17: Java에서 boolean 조건식을 쓸 때 주의할 점은?

조건식을 쓸 때는 반드시 논리연산자 `&&`, `||` 등을 명확히 써야 하며, `if` 안의 조건은 `boolean` 타입이 되어야 합니다.

또한 `0 < angle < 90` 같은 파이썬식 비교를 쓰면 `bad operand` 에러가 발생합니다.

## Q18: Java에서 변수 초기화를 꼭 해야 하나요?

Java에서는 지역 변수(local variable)는 **초기화하지 않으면 컴파일 에러**가 납니다. `int answer;`만 선언하고 값 할당 없이 사용하면 아래와 같은 에러가 발생합니다:

```text
variable answer might not have been initialized
```

해결 방법: 반드시 초기값을 줘야 함
```java
int answer = 0;
```

---

## Q19: 이 문제에서만 `int answer = 0;`을 꼭 써야 하나요?

그렇습니다. 조건 분기를 통해 모든 경우에 `answer`에 값이 할당되는 것이 보장되지 않는 구조라면, 컴파일러는 초기값이 없을 경우 에러를 냅니다.

즉, Java는 **초기값 없이는 사용할 수 없다**는 엄격한 규칙을 따릅니다.

## Q20: `= 0`은 숫자 0이라는 뜻이 아니라 초기화인가요?

`= 0`은 실제로는 '값을 0으로 초기화한다'는 의미입니다. 즉:

- `int answer = 0;` → `answer` 변수에 0이라는 값을 저장
- "초기화"는 변수 선언과 동시에 기본값을 주는 행위를 뜻함
- Java, Kotlin, C에서는 명시적으로 초기화하지 않으면 에러가 날 수 있음

즉, `= 0`은 단순한 0이 아니라 **초기 상태를 설정하는 역할**입니다.

## Q21: Kotlin에서 for문 문법이 왜 안 되는 건가요?

Kotlin에서는 C 스타일 `for (int i = 0; i < n; i++)`를 지원하지 않습니다.

### Kotlin에서의 기본 반복문 문법:
```kotlin
for (i in 0 until n) {
    ...
}
```

또는
```kotlin
for (i in 2..n step 2) {
    ...
}
```

### 요약 비교:

| 언어   | 문법 예시                      |
|--------|--------------------------------|
| C/Java | `for (int i = 0; i < n; i++)`  |
| Python | `for i in range(0, n)`         |
| Kotlin| `for (i in 0 until n)` 또는 `for (i in 2..n step 2)` |
