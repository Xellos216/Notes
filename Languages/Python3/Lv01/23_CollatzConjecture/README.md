# 23_CollatzConjecture

## 🧠 Problem
1937년 Collatz란 사람에 의해 제기된 이 추측은, 주어진 수가 1이 될 때까지 다음 작업을 반복하면, 모든 수를 1로 만들 수 있다는 추측입니다.

작업 규칙:
1. 입력된 수가 짝수라면 2로 나눕니다.
2. 입력된 수가 홀수라면 3을 곱하고 1을 더합니다.
3. 결과로 나온 수에 대해 같은 작업을 1이 될 때까지 반복합니다.

단, 작업을 500번 반복할 때까지 1이 되지 않으면 -1을 반환합니다.

### Example
- Input: `num = 6` → Output: `8` 
- Input: `num = 16` → Output: `4`
- Input: `num = 626331` → Output: `-1`

---

## 🐍 Python Reference Code

```python
def solution(num):
    for i in range(500):
        if num == 1:
            return i
        if num % 2 == 0:
            num = num // 2
        else:
            num = num * 3 + 1
    return -1
```

## ❓ Q & A

### Q1. 제한 사항에 왜 num은 1 이상 8,000,000 미만이라고 했나요?
- 이는 **테스트 환경을 안정적으로 유지하기 위한 조건**입니다.
- 예외적으로 너무 큰 값이 들어오면 계산량이 폭발하거나 무한 루프 위험이 있기 때문에, **문제 출제자가 사전에 범위를 제한**해둔 것입니다.
- 이 범위 안에서는 시간 제한, 메모리 제한 등을 초과하지 않고 문제를 해결할 수 있습니다.
- 따라서 **입력 값 범위는 조건에만 존재하고, 코드에 직접 처리하지 않아도 됩니다**.

---

### Q2. `range(500)`과 `range(1, 500)`의 차이는 뭔가요?
- `range(500)`은 `0`부터 `499`까지 **500회 반복**입니다.
- `range(1, 500)`은 `1`부터 `499`까지 **499회 반복**입니다.
- 즉, `range(500)`을 사용하는 것이 **정확히 500번 반복**을 보장하므로 이 문제에서는 적절합니다.
- 또한 `i`를 그대로 `return`할 때, 0부터 세는 `range(500)`이 더 일관된 로직이 됩니다.
