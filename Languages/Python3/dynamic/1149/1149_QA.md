## Q1: 문제 조건의 i번 집 색 조건이 직관적으로 이해되지 않았어요.

### 질문 내용:
```text
1번 집의 색은 2번 집과 같지 않아야 한다.
N번 집의 색은 N-1번 집과 같지 않아야 한다.
i(2 ≤ i ≤ N-1)번 집의 색은 i-1번, i+1번 집의 색과 같지 않아야 한다.
```

### 해설:
- 1번 집: 이웃인 2번 집과 색이 달라야 함
- N번 집: 이웃인 N-1번 집과 색이 달라야 함
- 중간 집들 (i번): 양 옆에 이웃이 2개 있으므로, 앞(i-1)과 뒤(i+1) 집 모두와 색이 달라야 함

### 비유:
학생들이 줄을 서서 옷 색을 고르는 상황이라면:
- 맨 앞은 뒷사람만 신경
- 맨 뒤는 앞사람만 신경
- 중간은 앞뒤 모두 신경  
→ 이 조건을 그대로 반영한 것이 `i-1`, `i+1`과 다른 색을 고르라는 의미임

## Q2: 프로그래밍 언어는 항상 0부터 시작하나요? [0] 혹은 [i-0] 이런 게 자주 보여요.

### 해설:

- 대부분의 프로그래밍 언어(Python, C, Java 등)에서 리스트/배열 인덱스는 **0부터 시작**함
- 이유는 메모리 주소 연산 때문: 시작 주소에 `+0`으로 첫 요소를 가리킬 수 있어서 효율적
- `cost[i][0]`: i번째 집의 빨강 비용을 의미함 (`0=빨강`, `1=초록`, `2=파랑`)
- `i - 0`은 사실상 i와 동일 → 가끔 연산 패턴상 쓰이지만 의미 변화는 없음

### 예외:
- MATLAB, Fortran 등의 일부 언어는 1부터 시작
