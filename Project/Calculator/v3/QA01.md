# 📘 CLI 계산기 실습 Q&A 정리 (비유 & 예시 포함)

---

### Q1: Enum이 뭐야?
**A:**  
Enum은 *서로 연관된 상수*들을 하나의 타입으로 묶은 열거형이다. Java에서는 `enum`도 내부적으로는 `class`처럼 동작하며, 타입 안정성을 제공하고 유지보수를 용이하게 한다. 예: `PLUS`, `MINUS`, `DIVIDE` 같은 연산자를 enum으로 정의 가능.

**💡 예시:** `public enum Direction { NORTH, SOUTH, EAST, WEST }`  
**🪑 비유:** 식당의 의자 종류를 `CHAIR_TYPE = { PLASTIC, WOODEN, METAL }`처럼 미리 정해둔 것.

---

### Q2: FIFO가 뭐지?
**A:**  
FIFO는 "First-In First-Out"의 줄임말로, **먼저 들어온 데이터가 먼저 나가는 구조**야. 줄을 서는 개념과 똑같아.

**💡 예시:** `Queue<String> q = new LinkedList<>(); q.add("A"); q.poll();`  
**🎫 비유:** 놀이공원 줄 서기! 먼저 줄 선 사람이 먼저 입장.

---

### Q3: 왜 빨간 줄이 보이는 거야?
**A:**  
enum 안에 람다식을 넣었지만, 그걸 받을 생성자나 필드가 없어서 컴파일 에러가 발생한 것.

**💡 예시:** 람다 `(a, b) -> a + b`를 받는 필드를 정의하지 않으면 오류  
**🧩 비유:** 레고 블록은 샀는데 끼울 구멍이 없어서 조립이 안 되는 상황.

---

### Q4: enum이 class야?
**A:**  
표현은 enum이지만, Java 내부적으로는 `Enum<T>`를 상속한 final class로 컴파일된다.

**💡 예시:** `enum Color { RED } → class Color extends Enum<Color>`  
**📦 비유:** 포장만 다를 뿐, 내부는 클래스 구조와 같다.

---

### Q5: 람다 연산이란?
**A:**  
연산을 `(a, b) -> a + b`처럼 함수처럼 전달하는 방식. enum과 함께 사용해 연산을 모듈화할 수 있다.

**💡 예시:** `BinaryOperator<Double> add = (a, b) -> a + b;`  
**🪄 비유:** 계산기를 직접 들고 하지 않고, 계산 방법만 전달하는 느낌.

---

### Q6: `ArithmeticCalculator calculator` 해석
**A:**  
`ArithmeticCalculator` 타입의 객체를 만들고, 그걸 `calculator`란 이름으로 참조하겠다는 뜻.

**💡 예시:** `Dog myDog = new Dog();`  
**🐶 비유:** 새 강아지를 입양하고 이름표 붙이기.

---

### Q7: `new Cup();`은 뭐야?
**A:**  
클래스 설계도 `Cup`을 기반으로 실제 객체를 메모리에 생성하는 명령.

**💡 예시:** `Cup cup = new Cup();`  
**🏗️ 비유:** 종이에 그린 설계도로 컵을 직접 만든 것.

---

### Q8: 오른쪽에서 왼쪽으로 읽는 거야?
**A:**  
실행 순서는 오른쪽 → 왼쪽(객체 생성 → 대입), 해석은 왼쪽 → 오른쪽이 자연스러움.

**💡 예시:** `int x = 5 + 3;`  
**🔄 비유:** 음식을 요리(오른쪽)한 다음 접시에 담기(왼쪽).

---

### Q9: Result, Max, Min, Sum은 무슨 의미?
**A:**  
`results` 리스트에 누적된 값으로부터 최근 결과, 최대값, 최소값, 총합을 구한 것.

**💡 예시:** `[21.0, 10.5] → Max = 21.0, Min = 10.5, Sum = 31.5`  
**📊 비유:** 가계부의 총지출, 최고지출, 최저지출 합계 같은 통계 요약.

---

### Q10: `fromSymbol()` 해석
**A:**  
문자열 기호(예: `"+"`)를 enum 상수로 변환해주는 정적 메서드.

**💡 예시:** `fromSymbol("+") → ArithmeticOperator.PLUS`  
**🎛️ 비유:** 스위치 `"+"`를 누르면 그에 맞는 계산 기능 작동.

---

### Q11: `safeGet()`이 뭐야?
**A:**  
예외가 날 수 있는 연산을 try-catch로 감싸서 `"Invalid input"` 같은 문자열로 안전하게 반환.

**💡 예시:** `safeGet(() -> calculator.getMaxResult())`  
**🛡️ 비유:** 에러가 나도 고장 안 나는 안전장치 달린 계산기.

---

### Q12: `supplier`은 되고 `Supplier`은 왜 안 돼?
**A:**  
`supplier`는 변수(객체) 이름이고, `Supplier`는 타입(인터페이스)이기 때문. 객체의 메서드는 객체 이름으로 호출해야 함.

**💡 예시:** `supplier.get(); // OK` vs `Supplier.get(); // 오류`  
**📘 비유:** "강아지 종류(Dog)"가 짖는 게 아니라, 실제 `myDog`가 짖는 것처럼.
