### Q1. `ArithmeticCalculator calculator = new ArithmeticCalculator();` 이 코드에서 의미는?

**답변:**  
이 코드는 `ArithmeticCalculator` 클래스의 인스턴스를 생성하고, 그 참조값을 변수 `calculator`에 저장하는 문장입니다. 자바에서는 `new` 키워드를 통해 객체를 메모리에 생성하고, 해당 객체의 메모리 주소가 변수에 담깁니다.

즉,
- `new ArithmeticCalculator()` → 객체 생성 (힙 영역에 메모리 할당)
- `ArithmeticCalculator calculator = ...` → 변수 선언 및 참조값 저장 (스택 영역에 주소 저장)

**예시:**
```java
class Cup {}

Cup myCup = new Cup(); // Cup 객체 생성 후 참조 변수에 저장
```

**실무 맥락:**  
서비스나 컨트롤러에서 특정 기능을 사용할 때, 클래스 인스턴스를 생성하고 메서드를 호출하는 기본 패턴입니다. 특히 의존성 주입(Dependency Injection)을 이해하기 위한 전단계로 중요합니다.

### Q2. 여기서 Run 버튼이 2개가 있는데 무슨 차이야?

**답변:**  
IntelliJ 기반 IDE에서는 클래스 또는 특정 메서드 단위로 실행이 가능한데, **녹색 ▶ Run 아이콘**은 실행 가능한 코드 위치마다 따로 표시됩니다.

- **클래스 왼쪽(녹색 Run 버튼)** → `Main` 클래스 전체 실행 (Java Application 실행 구성)
- **메서드 왼쪽(녹색 Run 버튼)** → 해당 `main` 메서드만 실행. 실제론 클래스 실행과 동일하지만 위치만 다름.

**결론:** 둘 다 사실상 `main()` 메서드를 실행하며 동작은 같고, 아이콘 위치가 다른 것뿐입니다.

**실무 맥락:**  
JUnit 테스트에서는 각 테스트 메서드 옆에 개별 Run 아이콘이 생기며, 메서드 단위로 실행할 수 있어 디버깅이나 리팩토링에 유용합니다.

### Q3. 연산에 `sqrt`를 추가했어. 근데 결과값이 자동으로 소숫점 12자리까지 나오는데, 이거 이유가 있는 거야?

**답변:**  
Java에서 `Math.sqrt()` 함수는 `double` 타입을 반환하며, 이 타입은 IEEE 754 기준의 **64비트 부동소수점**으로 약 **15~17자리의 정밀도**를 가집니다.

→ 출력 포맷을 지정하지 않으면 Java는 가능한 많은 자릿수를 그대로 보여줍니다.

**예시:**
```java
System.out.println(Math.sqrt(99));       // 출력: 9.9498743710662
System.out.printf("%.2f\n", Math.sqrt(99)); // 출력: 9.95
```

**실무 맥락:**  
금융/수학 연산에서는 반드시 포맷팅 필요 (예: `%.2f` 또는 `DecimalFormat` 사용). 사용자에게 표시되는 결과는 반드시 제어해야 합니다.

### Q4. 구조 정리를 하면서 이번에 `package`를 사용하게 되었어. 원리를 설명해줘.

**답변:**  
`package`는 자바에서 **클래스를 논리적으로 묶는 네임스페이스**입니다. 실제로는 클래스의 물리적 위치(폴더 구조)와 연동되어 작동합니다.

**구조 예:**
```java
package model;
public class ArithmeticOperator { ... }
```
→ 이 클래스는 `src/model/ArithmeticOperator.java`에 위치해야 함.

**import 관련:**
다른 패키지의 클래스를 쓰려면 반드시 `import` 선언이 필요함. 같은 패키지면 생략 가능.

**실무 구조 예:**
```
src/
├── model/
├── service/
├── ui/
├── util/
└── Main.java
```

**실무 목적:** 가독성 향상, 클래스 충돌 방지, 접근 제어 및 유지보수 편리

### Q5. 이항 연산과 단항 연산에 대해 설명해줘

**답변:**  
- **단항 연산자**: 피연산자 하나 → `-x`, `!flag`, `x++`
- **이항 연산자**: 피연산자 둘 → `x + y`, `a > b`, `a && b`

**네 코드에서:**
- `PLUS`, `MINUS` 등은 이항 연산
- `SQRT`는 단항 연산 → 그래서 `unary` flag를 enum에 둠

**실무 맥락:** 수식 파서나 계산기 엔진 구현 시, 단항/이항 구분은 토큰 파싱과 스택 처리의 핵심 조건이 됨.

### Q6. 개발에서 시그니처(Signature)가 무슨 의미야?

**답변:**  
시그니처란 **메서드 이름 + 파라미터 타입** 조합으로, 메서드를 구분하는 고유한 식별자입니다. **반환 타입은 포함되지 않음!**

**예:**
```java
void greet(String name);           // greet(String)
int sum(int a, int b);             // sum(int, int)
```

**반례 (오류):**
```java
int print(int x);
double print(int x); // ❌ 컴파일 에러! 시그니처 동일
```

**실무 맥락:** API 설계, 오버로딩, 리플렉션, 테스트 자동화 등에서 시그니처 기반으로 메서드를 식별하고 처리합니다.

### Q7. 연속계산, 괄호계산, 괄호+연속계산의 복잡도 차이가 뭐야?

**답변:**  

| 기능 유형            | 로직 복잡도 | 설명 |
|---------------------|--------------|------|
| 연속 계산만         | 낮음 ⭐      | 왼→오로 차례로 계산, 상태 추적 거의 없음 |
| 괄호 계산           | 중간 ⭐⭐     | 스택 사용, 괄호 열림/닫힘 추적 필요 |
| 괄호 + 연속 계산    | 복잡 ⭐⭐⭐⭐   | 중첩 구조 + 우선순위 처리 → 파서/트리 필요 |

**예:**
- `2 + 3 * 4` → 왼→오 계산이면 순서 틀림 (우선순위 고려해야)
- `(1 + 2) * (3 - 1)` → 괄호 먼저 처리
- `((1 + 2) * (4 - 3)) / (2 + 1)` → 중첩 괄호까지 포함되면 트리 구조 또는 후위표기법 필요

**실무 맥락:** 컴파일러, 수학엔진, 코드 평가기 등에서는 반드시 파서(parser) 또는 스택 기반 계산 로직 필요.
