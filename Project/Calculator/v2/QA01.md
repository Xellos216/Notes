# QA01.md

## Q1: JDK 설치 후 `java` 명령은 되는데 `javac` 명령은 안 될 때 원인?
- `JAVA_HOME` 환경 변수나 `PATH` 설정이 `jdk/bin`이 아니라 `jre/bin` 쪽을 가리킬 수 있음
- `javac`는 JDK에만 포함되므로 JRE만 설치되어 있는 경우도 발생

---

## Q2: IntelliJ에서 Java 프로젝트를 만들었는데 실행이 안 됨
- `src` 폴더 아래에 `package` 없이 클래스를 바로 만들 경우, IDE에서 기본 설정이 적용되지 않을 수 있음
- `main()` 메서드가 있는 클래스에서 `Shift + F10`으로 직접 실행하거나, Run Configuration을 수동 설정해야 함

---

## Q3: `javac Hello.java`로 컴파일했더니 `.class` 파일이 안 생김
- 컴파일 오류가 난 상태일 가능성 높음
- `javac` 실행 시 경고 외에 오류 메시지가 있는지 반드시 확인

---

## Q4: Java 실행 시 `Could not find or load main class` 에러가 뜰 때 원인?
- `java ClassName` 실행 시 `.class` 파일의 위치가 현재 디렉토리에 있어야 함
- `package`가 설정된 경우, 디렉토리 구조도 맞춰야 함
  예: `com.example.App` → `com/example/App.class`

---

## Q5: IntelliJ에서 프로젝트 내 클래스가 인식되지 않음
- `Project Structure` → `Modules`에서 소스 루트가 설정 안 됐을 수 있음
- `src` 폴더를 우클릭해서 "Mark Directory as → Sources Root"로 지정

---

## Q6: `.java` 파일 실행 시 GUI 창이 안 뜨는 경우
- GUI 이벤트 루프를 EDT(Event Dispatch Thread)에서 돌리지 않은 경우
- `SwingUtilities.invokeLater()`로 감싸야 안정적인 실행 보장

---

## Q7: 콘솔이 안 뜨고 프로그램이 바로 종료됨
- `main()` 함수에서 GUI 프레임을 만들었지만 `setVisible(true)`를 안 했을 가능성 있음
- 또는 `JFrame.EXIT_ON_CLOSE` 설정이 누락됨

---

## Q8: `javac`로 컴파일했는데 실행하면 `NoClassDefFoundError`
- `.class` 파일은 있지만, 클래스 경로 설정이 잘못되어 클래스 로더가 찾지 못하는 경우
- `java -cp . Hello` 같이 클래스패스 직접 지정해서 실행해볼 것

---

## Q9: `java App` 실행 시 `Exception in thread "main" java.lang.NoClassDefFoundError`
- 클래스 이름이 맞는지, `App.class`가 현재 경로에 있는지 확인
- `App` 클래스가 다른 클래스를 참조하는데 해당 클래스가 클래스패스에 없는 경우도 원인

---

## Q10: GUI 창의 크기가 너무 작아서 안 보임
- `setSize()` 또는 `pack()`을 호출해야 함
- `pack()`은 컴포넌트 크기에 맞춰 JFrame 크기를 자동 조정
