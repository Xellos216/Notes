# QA02.md

## Q11: IntelliJ에서 프로젝트가 너무 많아질 때 관리 방법?
- `.idea`와 `.iml` 파일들은 각각의 프로젝트 설정이므로 삭제 전 백업 필수
- 여러 프로젝트를 관리할 땐 `File → Project Structure`에서 SDK 설정 공통화

---

## Q12: `.jar` 파일 실행 시 더블클릭하면 안 열림
- GUI 앱이라도 `Main-Class`가 명시되어 있지 않거나, OS에서 `.jar`를 Java로 연결 안 했을 수 있음
- 터미널에서 `java -jar file.jar`로 실행해볼 것

---

## Q13: `.jar` 파일로 실행하면 클래스 찾을 수 없다고 뜸
- `MANIFEST.MF` 파일에 `Main-Class`가 누락되었거나, 경로가 잘못 지정된 경우
- `AppGUI`가 `src`에 있을 때는 `Main-Class: AppGUI` 로 지정해야 함

---

## Q14: IntelliJ에서 `.jar` 빌드했는데 실행 안 됨
- `Artifacts` 설정 시 Output Layout에서 클래스 파일이 정상적으로 포함됐는지 확인 필요
- `Project Structure → Artifacts → + → JAR → From modules with dependencies`로 다시 설정

---

## Q15: `java CalculatorV2GUI.jar` 실행 시 `ClassNotFoundException`
- `java CalculatorV2GUI.jar`는 잘못된 실행 방식 → `java -jar CalculatorV2GUI.jar`로 실행해야 함

---

## Q16: IntelliJ의 `META-INF` 폴더란?
- `.jar` 파일의 메타데이터를 담는 폴더
- `MANIFEST.MF`는 메인 클래스 지정, 클래스패스 설정 등의 정보를 담음

---

## Q17: `META-INF` 폴더는 삭제해도 되는가?
- `.jar` 파일 사용하지 않을 거면 프로젝트 내 `META-INF`는 삭제해도 무방함
- 하지만 다시 `.jar` 만들 생각 있다면 보관하는 게 좋음

---

## Q18: IntelliJ에서 생성된 `out/` 폴더는 무엇인가?
- 컴파일된 `.class` 파일과 빌드 산출물이 저장되는 디렉토리
- 자동으로 생성되며, 삭제해도 IDE가 다시 만듦

---

## Q19: `out/` 폴더는 삭제해도 되나?
- ✅ 삭제 가능함
- 빌드시 다시 자동 생성됨
- Git 사용 중이면 `.gitignore`에 등록하는 것이 일반적

---

## Q20: `.jar` 빌드 실패 시 어디서 로그를 확인해야 하나?
- IntelliJ 하단의 `Build` 탭 또는 `Event Log`에서 확인
- `File → Project Structure → Artifacts` 설정도 점검 필요
