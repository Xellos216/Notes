## Day6 QA 정리 - 백준 9184번 '신나는 함수 실행'

문제 유형: 동적 계획법 1 (재귀 + 메모이제이션)

---

### ✅ Q1. `sys.setrecursionlimit(100000)` 이 무슨 의미인가요?
- 파이썬은 기본적으로 재귀 함수를 1000번만 호출할 수 있도록 제한함.
- 하지만 이 문제는 재귀 깊이가 수만 번까지 깊어질 수 있으므로, `100000`으로 설정해 오류 방지.
- "재귀 호출을 최대 10만 번까지 허용해줄게" 라는 의미.

---

### ✅ Q2. `dp = [[[0]*21 for _ in range(21)] for _ in range(21)]` 이 코드가 뭔가요?
- `dp[a][b][c]` 구조의 3차원 리스트 생성 (크기 21x21x21)
- 이유: a, b, c는 0~20까지만 유효하므로 각 축에 대해 21개 크기로 설정
- 목적: 이미 계산된 `w(a, b, c)`의 결과를 저장해서 중복 계산을 줄이는 **메모이제이션 테이블** 역할

---

### ✅ Q3. `w(a, b, c)`는 파이썬 내장함수인가요?
- ❌ 아니다. 이 문제에서 **직접 정의한 사용자 함수**
- 파이썬에 내장된 함수는 아니며, `def w(a, b, c):`로 우리가 정의한 함수임

---

### ✅ Q4. `while True:`에서 왜 `map()`을 썼나요?
- 사용자 입력이 "1 2 3"처럼 한 줄로 들어오기 때문에, 이걸 나눠서 숫자로 바꾸기 위해 `split()`과 `map(int, ...)` 사용함
- `a, b, c = map(int, input().split())` → 입력값을 각각 정수로 저장하는 구조

---

### ✅ Q5. 예제 출력이 왜 그렇게 나오는지 모르겠어요
- 모든 출력은 `w(a, b, c)`에 정의된 조건과 재귀 구조에 따라 계산됨
- 예: `w(1, 1, 1)` → 조건 3에 해당되지 않음 → else문 실행 → 하위 함수로 분해되어 계산됨
- return 1 되는 시점은 `a <= 0 or b <= 0 or c <= 0`일 때임

---

### ✅ Q6. `dp`가 뭔가요?
- DP = Dynamic Programming (동적 계획법)
- 여기서 `dp`는 "이미 계산된 결과를 저장해두는 3차원 배열"
- 목적: 똑같은 `w(a, b, c)`를 다시 계산하지 않도록 하기 위함 (성능 향상)

---

### ✅ Q7. `w(50, 50, 50)` = 1048576인 이유는?
- 50은 모두 20보다 크므로 `w(20, 20, 20)`으로 자동 변환됨
- `w(20, 20, 20)`은 else문을 통해 수많은 재귀 호출이 발생
- 각 재귀 끝에서 `return 1`이 반복적으로 발생 → 그 합이 2^20 = 1048576

---

### ✅ Q8. 사용자 요약 검토 (50 50 50 → 1048576)
- 핵심 흐름은 정확히 이해함
- 보완: "1이 나왔다가 또 else로 가는 구조"는 아님 → 많은 재귀 호출의 끝에서 나오는 1값들이 모두 합쳐지는 구조임
