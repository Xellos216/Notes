## Q1. 지금 CLI로 MySQL 실습 중인데 DBeaver 같은 프로그램을 안 쓰고 CLI 환경에서 SQL을 실습했을 때 얻는 이점이 뭐야?

### ✅ A1.

CLI 환경에서 SQL을 실습하면 다음과 같은 장점이 있어:

1. **SQL 문법과 구조에 더 집중할 수 있음**  
   → GUI의 자동완성, 시각화 기능 없이 순수하게 SQL 문장 구성 연습이 가능.

2. **운영 환경과 유사한 경험**  
   → 실제 서버에서는 대부분 CLI 기반(MySQL Shell 또는 터미널 접속)으로 쿼리를 실행하므로 실무 감각과 가까움.

3. **스크립트 자동화/버전 관리에 유리함**  
   → SQL 문장을 `.sql` 파일로 관리하고 git 등으로 버전 관리 가능.

4. **리눅스 기반 도구와의 연계가 쉬움**  
   → Bash, `awk`, `sed`, `less` 등의 도구와 조합하여 처리 가능 (e.g. 백업, 로그 필터링 등)

## Q2. 나중에 테이블에 컬럼, 데이터들이 많아지면 그때는 CLI로 이걸 보는 게 훨씬 더 비효율적일 것 같은데 맞아?

### ✅ A2.

맞아, **데이터 규모가 커질수록 CLI는 시각적 탐색이나 테이블 구조 파악에 비효율적**일 수 있어. 그래서 일반적으로는 아래와 같은 분리 전략을 추천해:

- **학습/개발 초기:** CLI 중심  
  → SQL 로직 자체에 집중하고 기본 문법과 흐름 파악에 적합

- **실제 업무 or 복잡한 데이터 분석:** GUI 도구 병행 (예: DBeaver, DataGrip)  
  → 테이블 시각화, 쿼리 결과 정렬/필터링, 데이터 직접 수정 등 편의성 ↑

> 요약:  
> `CLI는 학습과 자동화 중심`,  
> `GUI는 시각적 분석과 탐색 중심`
