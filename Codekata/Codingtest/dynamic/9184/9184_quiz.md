### ✅ 퀴즈 및 정답

#### Q1. 다음 코드에서 `sys.setrecursionlimit(100000)`의 목적은 무엇인가요?
- A. 재귀 함수 실행을 막기 위해  
- B. 프로그램의 실행 시간을 제한하기 위해  
- C. 깊은 재귀 호출로 인한 오류를 방지하기 위해 ✅  
- D. 리스트의 최대 크기를 설정하기 위해  

> **해설**: 파이썬은 기본적으로 재귀 호출을 1000번만 허용하지만, 이 문제는 재귀가 매우 깊어질 수 있어 한도를 100,000까지 늘림

#### Q2. 다음 코드의 의미는?
```python
dp = [[[0]*21 for _ in range(21)] for _ in range(21)]
```
- A. 2차원 배열 생성  
- B. 3차원 배열 생성, w(a, b, c)의 중간값을 저장하기 위해 ✅  
- C. 21개의 1차원 리스트 생성 ❌  
- D. 리스트 안에 21개의 튜플 저장

> **해설**: a, b, c 각각 0~20까지만 유효하므로 21x21x21 크기의 3차원 배열 생성

---

#### Q3. `w(50, 50, 50)`의 출력값이 1048576이 되는 이유는?
- A. 50은 계산이 간단해서 바로 1048576을 반환한다  
- B. w(50, 50, 50)는 내부적으로 w(20, 20, 20)으로 바뀌고, 재귀 호출 끝에서 반환된 수많은 1이 더해져서 결과가 2^20이 된다 ✅  
- C. 1048576은 파이썬의 최대 재귀 깊이다  
- D. 입력값이 클수록 정해진 수학 공식으로 고정값이 나오기 때문이다

> **해설**: 문제 조건에 의해 a, b, c가 20을 초과하면 w(20, 20, 20)으로 계산됨. 그 결과가 2^20 = 1048576.

---

#### Q4. 재귀 함수 `w(a, b, c)`가 1을 반환하는 조건은?
- A. a == b == c  
- B. a > 20  
- C. a <= 0 or b <= 0 or c <= 0 ✅  
- D. a < b < c ❌

> **해설**: 0 이하인 값이 하나라도 있으면 1을 반환함.

---

#### Q5. `w(a, b, c)`가 이미 한 번 계산되었고, 그 결과가 저장되어 있다면 가장 효율적인 처리 방식은?
- A. 다시 계산을 시작한다  
- B. 저장된 값을 꺼내 사용한다 ✅  
- C. 입력을 무시한다  
- D. 프로그램을 종료한다

> **해설**: 메모이제이션의 목적은 중복 계산을 방지하고 이미 계산한 값은 재사용하는 것.
