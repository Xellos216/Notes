## Q45: 자릿수 합 구하기 문제, Python에서 어떻게 푸는지 감이 안 잡힘

정수 n의 자릿수를 모두 더하는 문제는 다음 두 가지 방법으로 해결 가능:

### 방법 1: 문자열 변환
```python
def solution(n):
    return sum(int(d) for d in str(n))
```

### 방법 2: 수학적 분해
```python
def solution(n):
    answer = 0
    while n > 0:
        answer += n % 10
        n //= 10
    return answer
```

## Q46: 자릿수 더하기 문제, C / Java / Kotlin 버전은?

### ✅ C
```c
int solution(int n) {
    int sum = 0;
    while (n > 0) {
        sum += n % 10;
        n /= 10;
    }
    return sum;
}
```

### ✅ Java
```java
public class Solution {
    public int solution(int n) {
        int sum = 0;
        while (n > 0) {
            sum += n % 10;
            n /= 10;
        }
        return sum;
    }
}
```

### ✅ Kotlin
```kotlin
class Solution {
    fun solution(n: Int): Int {
        return n.toString().map { it.digitToInt() }.sum()
    }
}
```

또는 수학적 방법:
```kotlin
class Solution {
    fun solution(n: Int): Int {
        var num = n
        var sum = 0
        while (num > 0) {
            sum += num % 10
            num /= 10
        }
        return sum
    }
}
```

## Q47: Kotlin에서 map { it.toString().toInt() } 안 되는 이유?

`it`은 `Char` 타입이므로 `.toString().toInt()`보다 `digitToInt()`를 쓰는 것이 더 안전하고 Kotlin스러움.

```kotlin
n.toString().map { it.digitToInt() }.sum()
```

## Q48: Kotlin에서 Solution 클래스 없다고 오류 나옴

테스트 시스템이 내부적으로 이렇게 호출하고 있음:

```kotlin
val sol = Solution()
sol.solution(123)
```

따라서 반드시 `fun solution()`을 `class Solution { ... }` 안에 넣어야 함.

## Q49: 약수의 합 (4개 언어 비교)

### ✅ Python
```python
def solution(n):
    return sum(i for i in range(1, n+1) if n % i == 0)
```

### ✅ C
```c
int solution(int n) {
    int sum = 0;
    for (int i = 1; i <= n; i++) {
        if (n % i == 0) sum += i;
    }
    return sum;
}
```

### ✅ Java
```java
public class Solution {
    public int solution(int n) {
        int sum = 0;
        for (int i = 1; i <= n; i++) {
            if (n % i == 0) sum += i;
        }
        return sum;
    }
}
```

### ✅ Kotlin
```kotlin
class Solution {
    fun solution(n: Int): Int {
        var sum = 0
        for (i in 1..n) {
            if (n % i == 0) sum += i
        }
        return sum
    }
}
```

## Q50: 나머지가 1이 되는 수 찾기 (4개 언어 비교)

### ✅ Python
```python
def solution(n):
    for x in range(2, n):
        if n % x == 1:
            return x
```

### ✅ C
```c
int solution(int n) {
    for (int x = 2; x < n; x++) {
        if (n % x == 1) return x;
    }
    return -1;
}
```

### ✅ Java
```java
public class Solution {
    public int solution(int n) {
        for (int x = 2; x < n; x++) {
            if (n % x == 1) return x;
        }
        return -1;
    }
}
```

### ✅ Kotlin
```kotlin
class Solution {
    fun solution(n: Int): Int {
        for (x in 2 until n) {
            if (n % x == 1) return x
        }
        return -1
    }
}
```

## Q51: Python에서 왜 `range(2, n)` 쓰는지?

- 문제의 제한사항 `n ≥ 3`은 **입력값 범위**
- `x`는 `n % x == 1`을 만족하는 가장 작은 자연수 → 최소값은 2부터
- 따라서 `range(2, n)`이 맞고, `range(3, n)`이면 정답을 놓칠 수 있음

## Q52: 왜 C, Java, Kotlin에는 `return -1`이 있는지?

- 이건 **방어용 코드**
- 실제로는 항상 해가 존재해서 실행되지 않지만,
- C/Java/Kotlin에서는 **모든 경로에서 return이 있어야** 컴파일 가능
- Python은 마지막 return이 없어도 오류 안 나므로 생략 가능
