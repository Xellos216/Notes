## Q33: 왜 홀수는 되고 짝수는 안 된 거지?

Python 코드에서 `//` 연산자는 **정수 나눗셈(몫)**입니다. 짝수 여부를 판단할 때는 `num % 2 == 0`처럼 **나머지 연산자 `%`**를 사용해야 정확히 판별됩니다.

```python
# 잘못된 코드
if num // 2 == 0:  # 짝수 판단 X

# 올바른 코드
if num % 2 == 0:
    return 'Even'
else:
    return 'Odd'
```

## Q34: C 코드에서 어디서 문법 오류가 난 거야?

1. `malloc(...)` 안에 크기 명시 안 함 → 오류 발생
2. `'Even'`은 `char` 하나만 나타내므로 `char*` 포인터에 넣을 수 없음 → `strcpy()`로 문자열 복사해야 함

### 수정 코드
```c
char* answer = (char*)malloc(6); // "Even" + '\0'
strcpy(answer, "Even");
```

## Q35: 여기서 `char`은 뭐고 `*`은 왜 붙는 거야?

- `char` → 문자 하나 (1바이트)
- `char*` → 문자열의 시작 주소를 가리키는 **포인터**
- `"Even"` 같은 문자열은 배열이므로 포인터로 다룸

## Q36: `malloc()`이 메모리 크기를 할당해주는 거 맞아? 왜 6?

맞습니다. `malloc()`은 지정한 바이트만큼 메모리를 할당합니다. `"Even"`은 4글자 + `\0` = 5바이트. 넉넉하게 6바이트 할당.

## Q37: 왜 `answer`는 괄호 없이 쓰고 `strcpy()`는 괄호가 있어?

- `answer`는 변수 → 괄호 없음
- `strcpy()`는 함수 → 괄호 + 인자 필요

```c
strcpy(answer, "Even");  // 함수 호출
```

## Q38: 자바에서 `return String answer;`는 왜 오류야?

`String answer`는 **선언문**이므로 `return`과 함께 쓸 수 없습니다.

→ 선언은 밖에서 하고, 값만 반환해야 합니다:

```java
String answer;
if (...) {
  answer = "Even";
}
return answer; // OK
```

## Q39: Kotlin에서 `var answer` 에러?

Kotlin에서는 **타입을 명시하거나, 값을 초기화**해야 변수 선언이 가능합니다.

```kotlin
var answer: String     // OK
var answer = "Even"    // OK
var answer             // ❌ 에러
```

## Q41: C에서는 평균 계산이 또 달라?

네. C에서는 `sum()`이 없기 때문에 **직접 for문으로 합을 구해야** 합니다.

```c
int sum = 0;
for (int i = 0; i < arr_len; i++) {
    sum += arr[i];
}
return (double)sum / arr_len;
```

---

## Q42: `for (size_t i = 0; i < arr_len; i++)`가 이해 안 돼

- `size_t i = 0`: 인덱스 변수 선언
- `i < arr_len`: 배열 길이만큼 반복
- `i++`: 한 번 돌 때마다 i 증가

→ 배열 전체를 순회하는 전형적인 패턴

---

## Q43: Java에서 평균 구하는 법

Java에는 `sum()` 함수 없음 → for문으로 합 구하기. 배열 길이는 `arr.length`.

```java
public double solution(int[] arr) {
    int sum = 0;
    for (int i = 0; i < arr.length; i++) {
        sum += arr[i];
    }
    return (double)sum / arr.length;
}
```

## Q44: Kotlin에서 for문이 이상해

- `for (int .. i)`는 Java 문법 → Kotlin에서는 아래처럼 써야 함:

```kotlin
fun solution(arr: IntArray): Double {
    var sum = 0
    for (i in arr.indices) {
        sum += arr[i]
    }
    return sum.toDouble() / arr.size
}
```
