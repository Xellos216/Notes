
# QA02.md – Java GUI & IntelliJ 관련 Q&A (프로젝트 구조, 오류 메시지 해석, GUI 구조 차이 등)

### Q12: `public class CalculatorGUI` 라고 적은 건데, 왜 에러가 나는 거야?

**에러 내용**  
> `class CalculatorGUI is public, should be declared in a file named CalculatorGUI.java`

**해설**  
Java에서는 **`public` 클래스는 반드시 파일 이름과 같아야 해**.  
- 클래스 이름이 `CalculatorGUI`면 파일 이름은 **`CalculatorGUI.java`**여야 한다.
- 지금은 `Main.java` 파일에 `public class CalculatorGUI`를 선언했기 때문에 컴파일 에러가 발생한 것이다.

**해결 방법**  
- 클래스 이름을 `Main`으로 바꾸거나,
- 파일 이름을 `CalculatorGUI.java`로 바꾸면 된다.

---

### Q13: IntelliJ에서 만든 src/Main.java가 문제야?

**정확히 말하면**  
- **IntelliJ가 문제는 아니고**, Java의 규칙 때문에 생긴 문제야.
- IntelliJ는 `Main.java`라는 이름의 파일을 만들었고, 너는 그 안에 `CalculatorGUI`라는 public 클래스를 썼기 때문에 에러가 난 것뿐이야.

**정리**  
- 클래스 이름과 파일 이름이 안 맞는 게 문제다.

---

### Q14: 그래픽 작업할 땐 보통 `main` 안에서 다 만들어?

**답변**  
- 보통은 **`main()` 메서드 안에서 프레임, 버튼, 이벤트 등 GUI 요소를 생성**하는 방식이 기본이야.
- 하지만 규모가 커지면 GUI 요소들을 **별도 클래스**로 분리해서 관리하기도 해. 예: `CalculatorFrame`, `ButtonPanel` 등.

### Q15: 버튼 누를 때마다 자동으로 값 계산되게 만드는 게 `addActionListener()`야?

**맞아!**  
`addActionListener()`는 버튼이 클릭될 때 실행할 코드를 지정하는 기능이야.

```java
calcButton.addActionListener(e -> {
    // 클릭 시 실행할 코드
});
```

### Q16: `Integer.parseInt()`에서 예외가 생기면 어떻게 돼?

**예외 상황**  
- 숫자가 아닌 문자열을 파싱하려고 하면 `NumberFormatException`이 발생해.

```java
String text = "abc";
int num = Integer.parseInt(text);  // 예외 발생!
```

**따라서**  
예외 처리를 해줘야 앱이 강제 종료되지 않아.

```java
try {
    int num = Integer.parseInt(text);
} catch (NumberFormatException e) {
    System.out.println("숫자가 아님!");
}
```

---

### Q17: `NumberFormatException e`는 그냥 임의로 쓴 거야?

**아니야!**  
- `NumberFormatException`은 **Java에서 제공하는 실제 예외 클래스**야.
- `e`는 보통 Exception 객체를 나타내는 변수명으로 자주 사용됨. (이건 관습이야)

---

### Q18: GUI에서는 `while(true)`가 왜 없어?

**CLI(콘솔)**에서는 사용자 입력을 계속 받기 위해 `while(true)`를 쓰는 경우가 많지만,  
**GUI 프로그램**에서는 이벤트 기반 구조이기 때문에 필요 없어.

- 사용자가 버튼을 클릭하면 `addActionListener()`가 실행됨.
- **무한 루프 대신 이벤트가 자동으로 처리**되는 구조야.

### Q20: GUI 코드엔 `while(true)`가 없는데 이유는?

같은 내용 반복이지만 요약하면:  
- GUI는 **이벤트 중심 구조**라서 무한 루프가 필요 없음.  
- CLI는 사용자 입력을 계속 받기 위해 루프가 필요했음.

---

### Q21: `switch(op)`의 `op`는 뭐야?

- `op`는 operator의 줄임말이야.
- `+`, `-`, `*`, `/` 중 선택된 연산자를 저장하고 있는 변수지.

---

### Q22: Java로도 이런 GUI를 많이 만들까?

**지금은 드물어.**  
- 예전에는 `Swing`, `AWT`로 GUI를 만들었지만,  
- 현재는 대부분 **웹 또는 모바일 앱으로 전환**되었어.

**실무에서 GUI Java는 다음에 주로 사용돼:**  
- **학교/학원 과제용**
- **레거시 유지보수용**
- **간단한 툴** (내부 자동화 도구 등)
