# QA01.md – Java CLI 기초 Q&A (Q1 ~ Q11)

---

### Q1: `class`는 뭐야?

- Java에서 **class(클래스)**는 객체를 만들기 위한 설계도야.
- 클래스 내부에는 변수(속성), 메서드(기능), 생성자 등을 정의할 수 있어.
- 예: `public class Main { ... }`는 `Main`이라는 이름의 클래스를 정의한 것.
- Java에선 모든 코드가 반드시 클래스 안에 포함되어야 해. (`class` 바깥에 함수만 따로 둘 수 없음)

---

### Q2: `public`은 뭐야?

- `public`은 **접근 제한자** 중 하나야.
- 클래스, 메서드, 변수에 붙어서 **다른 클래스나 외부에서도 접근 가능하도록** 만들어줘.
- 예: `public class Main`은 누구든지 이 클래스를 사용할 수 있다는 뜻이야.
- 반대로 `private`, `protected`는 제한된 접근만 가능해.

---

### Q3: `static`은 뭐야?

- `static`은 **클래스 소속**이라는 뜻이야. 객체를 생성하지 않고도 사용할 수 있어.
- `public static void main(...)`에서 `main`은 자바 프로그램의 시작점인데, 객체 생성 없이 바로 실행돼야 하니까 `static`이 꼭 필요해.
- 예: `Main.main()`처럼 바로 실행 가능.

---

### Q4: `void`는 뭐야?

- `void`는 **반환값이 없다는 의미**야.
- `public static void main(...)`에서 `main`은 프로그램을 실행시키기 위한 메서드일 뿐, 어떤 값을 돌려줄 필요가 없으니까 void를 사용해.

---

### Q5: `main(String[] args)`는 뭐야?

- `main()` 메서드는 Java 프로그램의 **시작 지점(Entry Point)** 이야.
- JVM이 프로그램을 실행할 때 제일 먼저 호출하는 메서드야.
- `String[] args`는 **커맨드라인 인자**를 받을 수 있는 배열이야.
- 예: `java Main hello world` → `args[0] = "hello"`, `args[1] = "world"`

---

### Q6: `Scanner`는 뭐야?

- `Scanner`는 **사용자 입력(키보드 입력)을 받을 때 사용하는 클래스**야.
- 보통 `System.in`과 함께 써서 표준 입력(키보드 입력)을 읽음.
- 예:
  ```java
  Scanner scanner = new Scanner(System.in);
  int num = scanner.nextInt();       // 숫자 입력
  String word = scanner.next();      // 단어 한 개 입력
  String line = scanner.nextLine();  // 한 줄 전체 입력
  ```

---

### Q7: 왜 `String`으로 입력받고 `int`로 바꿔?

- 사용자 입력은 항상 **문자열(String)** 형태로 들어와.
- 그래서 숫자 계산을 하려면 **문자열 → 숫자**로 바꿔야 해.
- `Integer.parseInt(input)`처럼 변환해줘야 실제 숫자 계산이 가능하지.

---

### Q8: `System.out.println()`은 뭐야?

- **표준 출력(콘솔 출력)**을 담당하는 명령어야.
- `System`: 자바 표준 클래스
- `out`: 출력 스트림 (콘솔용)
- `println()`: 괄호 안의 값을 출력하고 줄바꿈까지 해줌
- `print()`는 줄바꿈 없이 출력만 함.

---

### Q9: Java에서 쓰는 연산자엔 뭐가 있어?

| 종류 | 연산자 | 설명 |
|------|--------|------|
| 산술 | `+ - * / %` | 사칙연산, 나머지 연산 |
| 대입 | `= += -=` 등 | 변수에 값 저장 |
| 비교 | `== != < > <= >=` | true/false 결과 반환 |
| 논리 | `&& || !` | 논리합/논리곱/부정 |
| 기타 | `++ -- ?: instanceof` 등 | 단항 연산, 조건 연산 등 |

---

### Q10: `try`랑 `catch`는 왜 써?

- 프로그램 실행 중 **오류가 발생할 수 있는 부분을 감싸서 에러를 처리하는 문법**이야.
- `try { ... }` 안에는 예외가 발생할 가능성이 있는 코드
- `catch (Exception e) { ... }` 안에는 예외 발생 시 실행할 코드
- 예:
  ```java
  try {
      int num = Integer.parseInt("abc");
  } catch (NumberFormatException e) {
      System.out.println("숫자가 아닙니다!");
  }
  ```

---

### Q11: `Integer`는 뭐야?

- `Integer`는 **기본형 타입인 `int`를 객체처럼 다룰 수 있게 해주는 클래스**야.
- `Integer.parseInt(String)`처럼 문자열을 숫자로 바꾸는 기능도 제공해.
- `int`는 기본형(primitive type), `Integer`는 객체(object type)
- 둘 사이엔 자동 변환(auto-boxing/unboxing)이 가능함.
